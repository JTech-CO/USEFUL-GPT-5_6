# USEFUL GPT-5.6

> GPT-5.6을 더 정확하고 실용적으로 사용하기 위한 한국어 프롬프트 가이드

`USEFUL-GPT-5_6.MD`는 GPT에게 무조건 길고 복잡한 명령을 입력하는 방법을 설명하는 문서가 아닙니다.  
원하는 결과를 명확히 정의하고, 필요한 맥락과 출력 형식, 경계조건, 검증 기준을 전달하는 방법을 정리한 실전형 안내서입니다.

OpenAI의 공식 **Prompting** 문서를 바탕으로 핵심 원칙을 한국어로 재구성했으며, 일반적인 ChatGPT 대화부터 조사, 문서 작성, 파일 분석, 프로그래밍, Codex 작업까지 폭넓게 활용할 수 있도록 예시와 템플릿을 추가했습니다.

---

## 이 저장소는 무엇인가요?

이 저장소는 GPT-5.6을 사용할 때 참고할 수 있는 한국어 프롬프트 가이드 저장소입니다.

핵심 문서인 [`USEFUL-GPT-5_6.MD`](./USEFUL-GPT-5_6.MD)에는 다음 내용이 포함되어 있습니다.

- 효과적인 프롬프트의 기본 구조
- 목표, 맥락, 출력 형식, 경계조건을 작성하는 방법
- 일반 대화와 복합 작업의 차이
- 최신 정보 조사 및 파일 기반 분석 방법
- 문서 작성과 교정용 프롬프트
- Codex 코드 수정, 디버깅, 테스트, 리뷰 프롬프트
- UI 구현 및 리팩터링 작업 예시
- 실패하기 쉬운 프롬프트 패턴
- 바로 복사해 사용할 수 있는 한국어 템플릿
- 작업 완료 전 검증 체크리스트

이 저장소의 목적은 특정한 하나의 “만능 프롬프트”를 제공하는 것이 아니라, 사용자가 자신의 작업에 맞는 프롬프트를 직접 설계할 수 있도록 돕는 것입니다.

## 이 긴 MD 파일을 GPT에 전부 붙여넣어야 하나요?

**일반적으로는 전부 붙여넣을 필요가 없습니다.**

`USEFUL-GPT-5_6.MD`는 GPT에게 매번 입력하는 하나의 거대한 시스템 프롬프트가 아니라, 사람이 읽고 필요한 부분을 꺼내 사용하는 참고 문서에 가깝습니다.

가장 권장되는 사용 방법은 다음 세 가지입니다.

---

## 사용 방법 1. 전자책이나 매뉴얼처럼 읽기

가장 기본적인 사용 방법입니다.

처음부터 끝까지 한 번 읽거나, 필요한 항목만 찾아보는 방식으로 사용할 수 있습니다.

예를 들어 다음과 같은 상황에서 관련 장을 찾아보면 됩니다.

| 상황 | 참고할 내용 |
|---|---|
| GPT의 답변이 자꾸 엉뚱함 | 목표와 맥락 작성법 |
| 결과 형식이 매번 달라짐 | 출력 형식 지정법 |
| 존재하지 않는 정보를 만들어냄 | 경계조건과 검증 기준 |
| 긴 문서를 분석하고 싶음 | 파일 기반 분석 템플릿 |
| 최신 정보를 조사하고 싶음 | 최신 정보 조사 템플릿 |
| Codex로 버그를 수정하고 싶음 | 버그 수정 프롬프트 |
| 코드를 함부로 많이 바꿈 | 수정 범위와 보존 조건 |
| 테스트 없이 코드만 생성함 | 검증 및 테스트 지시 |

문서를 읽으면서 자신에게 필요한 프롬프트 구조를 익히는 방식입니다.

---

## 사용 방법 2. 필요한 템플릿만 복사하기

실제 사용에서는 이 방법이 가장 효율적입니다.

문서 전체를 복사하지 않고, 현재 작업에 맞는 템플릿만 가져온 뒤 내용을 채웁니다.

예를 들어 파일을 분석하려면 다음과 같은 템플릿을 사용할 수 있습니다.

```text
첨부 파일을 근거로 {분석 목적}을 수행합니다.

규칙:
- 파일에 없는 내용을 임의로 보충하지 않음
- 모든 핵심 수치와 주장을 원본 위치와 연결
- 사실, 해석, 가정을 구분
- 파일 간 충돌을 숨기지 않음
- 읽을 수 없거나 누락된 부분을 명시

출력:
1. 요약
2. 핵심 발견
3. 파일 간 일치·충돌
4. 위험 또는 누락 정보
5. 권고안
```

중괄호 부분과 세부 조건만 자신의 작업에 맞게 변경하면 됩니다.

## 사용 방법 3. ChatGPT 프로젝트나 지식 파일로 활용하기

반복적으로 이 가이드를 참고하게 하고 싶다면 `USEFUL-GPT-5_6.MD` 파일을 ChatGPT의 프로젝트, 파일 첨부 또는 사용자 지정 GPT의 지식 자료로 넣어 활용할 수 있습니다.

이 경우에도 파일의 모든 내용을 매 답변마다 강제로 따르게 하기보다는 다음과 같이 요청하는 편이 좋습니다.

```text
첨부된 USEFUL_GPT_5-6 문서의 원칙을 참고해 아래 작업용 프롬프트를 개선합니다.

목표:
기존 프롬프트의 모호한 부분, 누락된 경계조건, 검증 기준을 보완합니다.

원본 프롬프트:
{여기에 기존 프롬프트 입력}
```

또는 다음과 같이 사용할 수 있습니다.

```text
USEFUL_GPT_5-6 문서를 참고해 이 작업에 적합한 프롬프트 템플릿을 선택하고,
현재 요구사항에 맞게 완성된 프롬프트로 재작성합니다.
```

이 방식은 반복 작업이나 팀 단위 사용에 적합합니다.

## 권장하지 않는 사용 방법

### 문서 전체를 매번 채팅창에 붙여넣기

가능은 하지만 대부분의 경우 비효율적입니다.

문서 전체가 현재 작업과 직접 관련되지 않을 수 있으며, 실제 요청보다 참고 문서가 더 길어져 중요한 요구사항이 묻힐 수 있습니다.

### 모든 규칙을 한 프롬프트에 넣기

좋은 프롬프트는 규칙이 많아서 좋은 것이 아닙니다.

현재 결과에 영향을 주는 조건만 선택해야 합니다.

### 문서를 절대적인 정답처럼 사용하기

프롬프트 작성에는 하나의 고정된 정답이 없습니다.

간단한 질문은 짧은 문장 하나로도 충분하며, 복잡한 작업에서만 상세한 구조가 필요합니다.

### 템플릿을 수정하지 않고 그대로 사용하기

템플릿은 시작점입니다.

대상, 목적, 파일, 분량, 기술 스택, 금지 사항, 검증 기준을 실제 작업에 맞게 변경해야 합니다.

## 가장 간단한 활용법

아래 다섯 항목만 기억해도 대부분의 프롬프트 품질을 개선할 수 있습니다.

```text
[목표]
무엇을 완성해야 하는가?

[맥락]
어떤 상황과 자료를 참고해야 하는가?

[출력]
어떤 형식과 분량으로 내놓아야 하는가?

[경계조건]
무엇을 변경하거나 추측하면 안 되는가?

[검증]
완료 전에 무엇을 다시 확인해야 하는가?
```

예시:

```text
첨부한 회의록을 개발팀용 주간 보고서로 정리합니다.

결정 사항, 차단 요소, 다음 행동을 구분합니다.
각 행동 항목에는 담당자와 기한을 표시합니다.
회의록에 없는 내용은 추측하지 않습니다.

Markdown 형식으로 작성하고,
완료 전에 담당자나 기한이 빠진 항목을 별도로 확인합니다.
```

## 일반 프롬프트 예시

```text
아래 초안을 고객 안내문으로 교정합니다.

대상:
기존 서비스 이용자

목적:
서버 점검 일정 안내

출력:
- 한국어 존댓말
- 500자 이내
- 제목과 본문 구성
- 날짜, 시간, 문의처 유지

경계조건:
- 확정되지 않은 보상 정책 추가 금지
- 원문에 없는 장애 원인 추측 금지

완료 전에 날짜와 시간이 원문과 일치하는지 확인합니다.
```

## 이 저장소가 다루지 않는 것

이 저장소는 다음 내용을 직접 제공하지 않습니다.

- 특정 업무를 위한 완성형 자동화 시스템

- 모델의 내부 추론 과정
- API 호출 코드 전체
- 특정 조직의 보안·법무·감사 기준
- 프롬프트만으로 보장되는 100% 정확한 결과

중요한 업무에서는 GPT의 결과를 사람이 검토해야 하며, 최신 정보·법률·의료·금융·보안과 관련된 내용은 신뢰할 수 있는 출처와 전문가 검증이 필요합니다.

## 문서의 핵심 철학

좋은 프롬프트는 길어서 좋은 것이 아닙니다.

좋은 프롬프트는 다음이 명확합니다.

- 무엇을 완성해야 하는지
- 어떤 자료를 사용해야 하는지
- 어떤 형태로 결과를 내야 하는지
- 무엇을 하면 안 되는지
- 성공 여부를 어떻게 확인할지

GPT에게 모든 사고 과정을 통제하려 하기보다, 결과와 성공 조건을 명확히 정의하는 것이 핵심입니다.
